Q：查看文件的两个版本之间的差异如何帮助你了解引入的错误？
A：通过版本之间的不同，发现typo等改动中无意引入的错误。
Q：从长远来看，能轻松访问文件的整个历史记录将如何提高编程者的效率？
A：轻松返回某个版本，以及发现错误。
Q：你认为，手动选择何时创建提交（像在 Git 中做的那样）与自动保存版本（如 Google Docs 所做的那样）各有何优缺点？
A：手动方便控制提交的时点，保证每次提交都有意义。自动可以防止忘记及时提交更改。
Q：何你认为一些版本控制系统（例如 Git）允许在一次提交中保存多个文件，而其他系统（例如 Google Docs）单独对待每个文件？
A：面向的对象不同。Git主要面向程序员，其他系统主要面向一般用户。
Q：如何使用命令 git log 和 git diff 来查看文件的历史记录？
A：在repository目录下输入git log可查看文件的历史记录，输入q退出。输入git diff 老版本ID 新版本ID，可比较文件更改。
Q:使用版本控制，如何让你更有信心进行有可能会造成破坏的更改？
A:随时可以返回更改前的任意版本，因此有信心进行可能会造成破坏的更改。
Q:现在你已设置了工作空间，你想尝试使用 Git 来做些什么？
A:版本管理。
